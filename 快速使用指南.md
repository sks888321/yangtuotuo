# 快速使用指南

## 启动项目

```bash
# 安装依赖（首次运行）
pnpm install

# 启动开发服务器
pnpm dev
```

## 首次使用

1. **选择存储目录**（可选）
   - 如果浏览器支持 File System Access API，系统会提示选择存储目录
   - 选择一个空文件夹用于存储数据
   - 数据将以 JSON 文件形式保存

2. **如果跳过或浏览器不支持**
   - 系统自动使用 localStorage 作为存储
   - 功能完全正常使用

## 核心功能使用

### 查看本周课程 📅

1. 进入"仪表盘"或"排课管理"页面
2. 查看周课程表，可以：
   - 看到整周的课程安排
   - 不同颜色代表不同课程
   - 点击课程查看详情
   - 使用箭头切换周次

### 快速排课 ⚡

**方法1：直接点击时间槽**
1. 在周课程表上点击空白的时间槽
2. 系统自动填充日期和时间
3. 选择课程和学生
4. 保存

**方法2：手动添加**
1. 点击"添加排课"按钮
2. 填写完整信息
3. 保存

### 管理用户 👥

1. 进入"用户管理"
2. 点击"添加用户"
3. 填写信息，选择角色（学生/教师/管理员）
4. 保存

### 添加课程 📚

1. 进入"课程管理"
2. 点击"添加课程"
3. 选择授课教师
4. 填写课程详情（名称、价格、时长等）
5. 保存

### 记录缴费 💰

1. 进入"缴费管理"
2. 点击"添加缴费"
3. 选择学生和课程
4. 填写金额
5. 保存

## 数据存储位置

### 如果选择了文件存储
数据保存在您选择的文件夹中：
- `users.json` - 用户数据
- `courses.json` - 课程数据
- `schedules.json` - 排课数据
- `payments.json` - 缴费数据

### 如果使用 localStorage
数据保存在浏览器缓存中，可以在浏览器开发者工具中查看。

## 数据备份

**文件存储方式**：
- 直接复制存储文件夹即可

**localStorage方式**：
- 使用浏览器的导出功能
- 或者切换到文件存储方式

## 常见问题

**Q: 数据会丢失吗？**
A: 使用文件存储不会丢失；localStorage 清除浏览器缓存会丢失。

**Q: 可以在多个设备上使用吗？**
A: 可以复制存储文件夹到其他设备使用。

**Q: 如何重新选择存储目录？**
A: 刷新页面，系统会重新提示选择（或清除 IndexedDB）。

**Q: 浏览器不支持文件API怎么办？**
A: 系统会自动使用 localStorage，功能完全正常。

## 技巧提示

1. **周课程表**：按住 Ctrl 滚动可以缩放（取决于浏览器）
2. **搜索**：在用户和课程管理页面都有搜索功能
3. **筛选**：排课列表可以按日期筛选
4. **状态管理**：及时更新课程状态为"已完成"或"已取消"
5. **教室信息**：添加排课时可以记录教室位置

## 浏览器要求

**推荐浏览器**（支持文件存储）：
- Chrome 86+
- Edge 86+
- Opera 72+

**其他浏览器**：
- Firefox（使用 localStorage）
- Safari（使用 localStorage）

所有现代浏览器都可以正常使用，只是存储方式不同。
